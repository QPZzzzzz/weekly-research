# build-acceleration — Research Memory

最后更新: 2026-06-04

### 关键记忆点提取

#### 公司/产品/项目
- **百度**：分布式编译加速方案，整合空闲资源并行编译。
- **美团**：编译耗时优化，45分钟→8分钟（分布式+缓存）。
- **腾讯**：yadcc，支持512并发编译。
- **Celonis**：homcc，低带宽（<10 Mbps）场景优化。
- **Incredibuild**：平台级基础设施，被微软VS 2026集成。
- **Meta**：Buck2开源，核心与规则分离。
- **sccache**：与mold链接器兼容性问题（GitHub Issue #1755）。
- **ccache**：缓存工具，生态活跃。
- **CMake**：仍占主导，但Bazel/Buck2挑战。

#### 重要趋势信号
- **AI深度集成CI/CD**：预测性缓存和自愈管道成为标配（强度：high）。
- **分布式编译+缓存组合**：工业级标配，百度、美团、homcc验证（强度：high）。
- **缓存工具兼容性痛点**：sccache+mold问题凸显工具链测试薄弱（强度：high）。
- **CI/CD 10分钟规则**：行业共识，倒逼激进加速策略（强度：high）。
- **Incredibuild平台化**：从工具商升级，挤压独立工具（强度：high）。
- **homcc低带宽场景**：新信号，满足远程办公需求（强度：medium）。
- **现代构建系统挑战**：Bazel/Buck2功能强但易用性门槛（强度：medium）。

#### 值得长期跟踪的技术方向
- **AI驱动CI/CD优化**：预测性缓存、自愈管道。
- **分布式编译+缓存组合**：低带宽场景（homcc）和工业级实践。
- **缓存工具兼容性**：sccache、mold等工具链集成测试。
- **现代构建系统**：Bazel、Buck2 vs CMake的演进。
- **Incredibuild平台化**：对独立工具（distcc、yadcc）的竞争影响。

#### 竞品动态
- **新产品**：Celonis开源homcc（低带宽分布式编译）。
- **融资/合作**：Incredibuild被微软VS 2026集成。
- **技术突破**：百度分布式编译方案、美团45→8分钟案例、腾讯yadcc 512并发。
- **兼容性问题**：sccache+mold（Issue #1755）暴露工具链集成风险。
