# build-acceleration — Research Memory

最后更新: 2026-06-02

### 公司/产品/项目
- **微软**: Visual Studio 2026, GitHub Copilot 构建性能分析
- **Incredibuild**: 被 VS 2026 集成，升级为平台级基础设施
- **腾讯**: yadcc (开源，支持 512 并发编译)
- **美团**: 分布式编译+缓存案例 (45分钟→8分钟)
- **百度**: 分布式编译方案理论文章
- **缓存工具**: ccache, sccache, zccache (新进入者)
- **链接器**: mold (与 sccache 有兼容性问题)
- **构建系统**: Bazel, Buck2 (易用性挑战), CMake (传统主导)

### 重要趋势信号
- **AI 深度集成至开发工具链** (方向: up, 强度: high): VS 2026 集成 Copilot 构建性能分析，自动检测编译瓶颈并推荐优化策略。
- **分布式编译+缓存组合成标配** (方向: up, 强度: high): 美团案例 (45→8分钟)、百度文章、腾讯 yadcc 验证工业级价值。
- **缓存工具生态空前活跃，竞争加剧** (方向: up, 强度: high): ccache 迭代，sccache 受关注，zccache 出现，但兼容性问题 (sccache + mold) 成为新痛点。
- **AI 驱动 CI/CD 管道成为趋势** (方向: up, 强度: high): 2026 年指南强调 AI 驱动优化、预测性缓存和自愈管道。
- **Incredibuild 从独立工具商升级为平台级基础设施** (方向: up, 强度: high): 被微软官方集成，将极大降低使用门槛，挤压同类工具市场。
- **C++26 新特性增加编译复杂度** (方向: up, 强度: medium): 反射等“史诗级”特性可能显著增加编译时间，反向催生优化需求。
- **现代构建系统挑战传统，但易用性仍是门槛** (方向: stable, 强度: medium): Buck2 开源，Bazel 功能强，但安装复杂、Windows 支持不佳，传统方案仍占主导。

### 值得长期跟踪的技术方向/话题
- AI 从“辅助编码”转向“优化构建”的落地效果与工具链集成。
- 分布式编译+缓存组合策略的最佳实践与工具兼容性 (如 sccache + mold)。
- C++26 新特性对编译性能的量化影响及针对性优化工具。
- Incredibuild 平台化后对独立工具 (如 distcc, yadcc) 市场空间的挤压。
- 新缓存工具 zccache 的差异化功能与生态定位。

### 竞品动态
- **新产品**: zccache (新缓存工具)；腾讯 yadcc (开源，512 并发)。
- **融资/合作**: Incredibuild 被微软 Visual Studio 2026 官方集成 (合作升级)。
- **技术突破**: 微软 VS 2026 集成 Copilot 构建性能分析 (AI 优化)；美团案例实现 45→8 分钟构建 (分布式+缓存组合)。
- **兼容性问题**: sccache 与 mold 链接器存在兼容性问题 (GitHub Issue #1755)，提示工具链集成测试重要性。
