# incredibuild — Research Memory

最后更新: 2026-05-21

- **公司/产品/项目：** Incredibuild、EngFlow、FASTBuild、Bazel、CMake、Xmake、Azure DevOps、Jenkins、GitLab、Visual Studio、龙智、Uber。
- **重要趋势信号：**
    - **Incredibuild 战略转型**：从单一编译工具转向“AI 驱动的 SDLC 平台”，方向：产品升级，强度：high。
    - **EngFlow 威胁升级**：以“性能（21倍加速）+ 安全（SOC 2）”组合拳直接挑战 Incredibuild 核心市场，方向：竞品威胁，强度：high。
    - **开源/CI/CD 工具分流**：FASTBuild+CMake 集成、Xmake 内置分布式编译、以及 Azure DevOps 等通用工具成为替代方案，方向：生态竞争，强度：medium。
    - **Incredibuild 技术声誉风险**：社区质疑其系统调用捕获和预处理器实现，方向：产品信任，强度：low。
    - **Incredibuild 中国渠道推广**：龙智作为授权合作伙伴，主攻游戏开发（UE4）场景，方向：区域市场，强度：low。
- **长期跟踪技术方向：**
    - AI 驱动的软件开发生命周期（SDLC）平台。
    - 分布式编译与远程执行（RBE）的性能与安全平衡。
    - 开源构建系统（如 FASTBuild、Bazel）与 CMake 的生态整合。
    - 跨平台构建工具（如 Xmake）的分布式编译能力。
- **竞品动态：**
    - **EngFlow**：获得 SOC 2 安全认证，宣称 C++ 构建速度提升 21 倍，与 Uber 等大客户合作。
    - **FASTBuild**：获得 CMake 官方集成支持，提供分布式和缓存构建方案。
    - **Xmake**：内置分布式编译功能，作为新兴跨平台工具蚕食用户。
    - **Incredibuild**：发布 AI 平台，覆盖 SDLC 全流程；修复 10.34.2 版本性能问题；与 Visual Studio 17.6 深度集成。
