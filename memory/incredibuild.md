# incredibuild — Research Memory

最后更新: 2026-06-06

### 公司/产品/项目
- **Incredibuild**: 核心公司，正从编译加速工具向AI全生命周期平台转型。
- **EngFlow**: 竞品，与tipi.build合作推出CMake RBE方案，正面竞争。
- **FASTBuild, Bazel**: 开源工具，持续分流用户但企业级支持不足。
- **tipi.build**: 与EngFlow合作，切入C++市场。
- **Microsoft, Visual Studio**: 深度集成Incredibuild，巩固生态。
- **龙智**: Incredibuild中国授权合作伙伴。

### 重要趋势信号
- **Incredibuild 向 AI 全生命周期平台转型** (high): 2025年9月发布AI平台，覆盖开发到部署，已有22万用户。
- **构建缓存成为核心功能，共享缓存集群提升效率** (high): Incredibuild 10.22新增共享缓存，微软强调其减少团队构建时间。
- **安全合规成为企业选型关键，Incredibuild 强调 SBOM 生成** (high): 官网突出安全供应链和防篡改SBOM，成为准入门槛。
- **EngFlow 与 tipi.build 联合推出 CMake RBE 方案，正面竞争** (medium): 直接对标Incredibuild的C++市场，宣称分钟级构建。
- **Visual Studio 深度集成 Incredibuild，巩固微软生态** (medium): VS 17.6集成Incredibuild 10，官方推广。
- **开源工具 FASTBuild 与 Bazel 持续分流用户** (medium): 用户关注度稳定，但企业级能力不足。
- **C++26 标准发布推动性能与安全升级，刺激编译需求** (medium): 契约机制和开发者增长，推动存量项目升级。

### 值得长期跟踪的技术方向或话题
- **AI驱动的全生命周期平台**: Incredibuild转型方向，提升用户粘性。
- **共享缓存集群**: 从算力分发转向数据复用，可能成为行业标配。
- **安全合规与SBOM**: 企业选型关键门槛，Incredibuild率先产品化。
- **CMake RBE方案**: EngFlow与tipi.build合作，测试市场接受度。
- **C++26新特性对编译需求的影响**: 契约机制等可能拉动编译加速工具需求。
